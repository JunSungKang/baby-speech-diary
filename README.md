# 📖 아기 발음 기록장 - 책 레이아웃 구현 완료

## 🎯 구현 개요

기존 스크롤 기반 카드 레이아웃을 **실제 책을 넘기는 경험**으로 전면 재구성했습니다.
이 레이아웃은 웹 환경에서 책을 읽는 UX를 재현하며, 향후 **인쇄용 책 제작**까지 염두에 둔 구조입니다.

## ✨ 주요 특징

### 1. 책 형태 레이아웃
- **한 화면 = 책의 한 펼침** (좌측 페이지 + 우측 페이지)
- 스크롤이 아닌 **페이지 전환**으로 콘텐츠 탐색
- 스프링노트 스타일의 중앙 제본 시각화

### 2. 페이지 구조

#### 좌측 페이지
- 📷 **사진 삽입 영역** (4:3 비율 고정)
- 📝 **메모 영역** (선택적)
- 📄 **페이지 번호** (짝수)

#### 중앙 제본
- 🔘 **스프링 링 12개** (균등 배치)
- 그라디언트 배경으로 입체감 표현

#### 우측 페이지
- 📚 **단어 목록** (페이지당 정확히 10개)
- 발음 단계별 계층 구조:
  ```
  1. 특정 단어
   → 처음: 첫 발음
   → 발전: 발전된 발음 1
   → 발전: 발전된 발음 2
  ```
- 📄 **페이지 번호** (홀수)

### 3. 디자인 요소
- ✅ **모눈종이 배경** (1cm 단위 그리드)
- ✅ **종이 질감** (과도하지 않은 텍스처)
- ✅ **제한적 색상 사용** (가독성 우선)

### 4. 인터랙션

#### 키보드
- `→` (오른쪽 화살표): 다음 펼침
- `←` (왼쪽 화살표): 이전 펼침
- `Home`: 첫 페이지
- `End`: 마지막 페이지

#### 마우스
- **좌측 페이지 클릭**: 이전 펼침
- **우측 페이지 클릭**: 다음 펼침
- **호버 효과**: 커서 변경 + 그림자 강조

#### 버튼
- 하단 "이전/다음" 버튼으로도 전환 가능

### 5. 페이지 전환 애니메이션
- 3D 페이지 넘김 효과 (CSS transform)
- 지속 시간: 600ms
- GPU 가속 최적화

```

├── assets/
│   ├── css/

│   │   └── book.css          # 책 레이아웃 전용 스타일 ⭐
│   └── js/
│       └── book.js           # 페이지 전환 로직 ⭐
├── index.md                  # 메인 페이지 (카드 레이아웃)
├── book.md                   # 책 레이아웃 페이지 ⭐
└── book-demo.html            # 독립 실행형 데모 ⭐
```

## 🚀 사용 방법

### 방법 1: Jekyll 서버 실행 (권장)

```bash
# Jekyll 설치 (Ruby 필요)
gem install bundler jekyll

# 프로젝트 디렉토리로 이동
cd baby-speech-diary

# 의존성 설치
bundle install

# 로컬 서버 실행
bundle exec jekyll serve

# 브라우저에서 접속
# http://localhost:4000/baby-speech-diary/book.html
```

### 방법 2: 독립 실행형 데모 (빠른 테스트)

```bash
# 간단한 HTTP 서버 실행 (Python 필요)
cd baby-speech-diary
python -m http.server 4000

# 브라우저에서 접속
# http://localhost:4000/book-demo.html
```

또는 `book-demo.html` 파일을 브라우저에서 직접 열기 (파일 경로로 접근)

## 📱 반응형 디자인

### 지원 화면 크기
- ✅ **데스크톱**: 1200px 이상 (최적)
- ✅ **태블릿**: 768px ~ 1199px (비율 유지 축소)
- ⚠️ **모바일**: 768px 미만 (안내 메시지 표시)

### 모바일 대응
768px 미만 화면에서는 다음 메시지가 표시됩니다:
> "이 서비스는 태블릿 이상 화면에 최적화되어 있습니다."

**설계 철학**: 책 UX는 물리적 제약을 모방하는 것이므로, 모바일에서 억지로 구현하면 UX가 훼손됩니다. 명확한 경계 설정이 더 나은 선택입니다.

## 🎨 커스터마이징

### 페이지당 단어 수 변경
`assets/js/book.js` 파일에서:
```javascript
wordsPerPage: 10,  // 원하는 개수로 변경 (8, 10, 12 권장)
```

### 색상 변경
`assets/css/book.css` 파일의 `:root` 섹션에서:
```css
:root {
    --progress-fill: #4A90E2;  /* 진행률 바 색상 */
    --text-primary: #222222;   /* 주요 텍스트 색상 */
    /* ... 기타 색상 변수 */
}
```

### 애니메이션 속도 조정
```css
:root {
    --transition-duration: 600ms;  /* 원하는 속도로 변경 */
}
```

## 🖨️ 인쇄 기능

### 브라우저 인쇄
1. 책 레이아웃 페이지 열기
2. `Ctrl + P` (Windows) 또는 `Cmd + P` (Mac)
3. 용지 크기: **A4 가로**
4. 여백: 없음
5. 배경 그래픽: **활성화**

### PDF 저장
브라우저 인쇄 대화상자에서 "PDF로 저장" 선택

### 인쇄 최적화
- 모눈종이 배경이 인쇄됨 (300dpi 권장)
- 페이지 네비게이션 바 및 버튼은 자동 숨김
- 각 펼침이 한 페이지로 출력됨

## 🔧 기술 스택

- **HTML5**: 시맨틱 마크업
- **CSS3**: Flexbox, Grid, CSS Variables, Animations
- **JavaScript (Vanilla)**: 페이지 전환 로직
- **Jekyll**: 정적 사이트 생성 (선택적)

## 📊 성능 최적화

- ✅ **GPU 가속**: `transform3d`, `will-change` 사용
- ✅ **디바운스**: 300ms 클릭 방지
- ✅ **애니메이션 중 입력 차단**: `pointer-events: none`
- ✅ **최소 리플로우**: 동적 렌더링 최적화

## 🐛 알려진 제약사항

### 1. 발음 단계 과다 시
- 발전 발음 4개 이상: 폰트 크기 자동 축소 (13px)
- 발전 발음 5개 이상: 폰트 크기 추가 축소 (12px)
- 그래도 넘칠 경우: 말줄임(...) + 툴팁 (향후 구현)

### 2. 모바일 미지원

- 별도 모바일 버전 필요 시 단일 페이지 스크롤 모드 구현 필요

### 3. 브라우저 호환성
- 최신 Chrome, Firefox, Safari, Edge 지원
- IE11 미지원 (CSS Grid, CSS Variables 사용)

## 📝 데이터 형식

### 마크다운 테이블 형식
```markdown
| 발음/단어 | 상황 및 의미 ||
|:---:|:---|:---|
| 특정단어 | 처음발음 ||
| 특정단어2 | 처음발음 | 발전발음1 |
| 특정단어3 | 처음발음 | 발전발음1 | 발전발음2 |
```

- 첫 번째 열: 특정 단어
- 두 번째 열: 처음 발음
- 세 번째 열 이후: 발전된 발음들 (가변 개수)
